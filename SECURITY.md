# 安全策略

## 支持的版本

| 版本 | 支持状态 |
| --- | --- |
| V11.6 | ✅ 积极维护 |
| < V11.6 | ❌ 不再维护 |

## 报告漏洞

我们非常重视安全问题。如果你发现了漏洞哨兵本身的安全漏洞,请**不要**通过公开 issue 报告,而是私下发邮件给维护者。

报告内容请包含:

1. 漏洞的简要描述
2. 复现步骤
3. 潜在影响
4. 你的联系方式(便于我们确认修复方案)

我们承诺在收到报告后 48 小时内回复,并在确认后尽快发布修复。

## 已知安全限制

- 漏洞哨兵只检测**已授权**网站。扫描未授权网站可能触犯法律。
- `JWT_SECRET` 默认值为空,生产环境部署前**必须**修改。
- `LLM_API_KEY` 会写入配置,但启用 `repr=False` 防止日志泄露。仍建议使用 secret 管理器而不是 `.env`。
- 自动巡检功能会以系统身份访问外部站点,生产环境请评估合规风险。

## 依赖安全

我们使用 GitHub Dependabot(配置在 `.github/dependabot.yml`)自动检测依赖漏洞。
